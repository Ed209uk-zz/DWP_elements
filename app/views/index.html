DWP internal elements guide

This guide explains how to use elements for internal DWP services and services that aren’t hosted on a GOV.UK domain.

You should use these elements in addition to the standard GOV.UK design elements.

They should be used when you’re working on:

  • a staff-facing DWP service hosted on GOV.UK (this could be the staff interface of a public-facing service)
  • a public or staff-facing DWP service that isn’t hosted on GOV.UK

Functionality in the top bar

There shouldn’t be any interactive elements in the top bar. Research by GDS and others shows that users find it difficult to see them.

Alpha and beta banners

Only use a banner if the service is part of a Digital-by-Default service.

Naming internal services

The name of an internal system should be consistent with the name of the public-facing service it supports. This could be the operational service function like ‘Universal Credit’ or a verb that states the core function of a standalone tool like ‘(Make a) Real Time Information request’.

Footer

Version 1 - for a public-facing service not on GOV.UK

This footer is similar to the standard one. It should always say: ‘Built by DWP Digital’. Work with your service team to decide which other links to include.

It’s not necessary to display a footer on staff-facing services but you may find it useful. Don’t display a footer on staff-facing services.

Typography

Font

GDS Transport Website (light and bold) - for a staff-facing service on GOV.UK

Noto Sans - for any service that isn’t on GOV.UK

This is a free, open source font. We can’t use GDS Transport Website because of licensing restrictions.

Colour

Use the extensive GOV.UK colour palette for all services, whether public or staff-facing. You can reduce the strength of colours by using tints of 50% and 25%.

Layout

Use the standard GOV.UK layout guidelines wherever possible. Consider whether you can design a better experience by using the full width of the screen eg for a staff-facing service that will only be used on a wide-screen monitor. Use the menu pattern in the DWP pattern library for sub-navigation in a staff-facing service.

Icons

Use icons sparingly on public-facing services. You may find them more useful on staff-facing services. Icons must be consistent across services. More guidance around icons will be created if necessary.

Form elements

Use the GOV.UK guidelines for all services.

Consider that forms within staff interfaces are likely to be used repetitively, every day. You may find it’s necessary to use non-standard elements such as drop-down lists. Test these thoroughly before implementing them.

Don’t assume that staff will understand your interface the first time they see it. Not everyone will be a ‘power user’.

Errors and validation

Use the standard guidelines in the GOV.UK elements library.
